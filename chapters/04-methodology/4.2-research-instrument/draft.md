## 4.2 Research Instrument or Sources of Data

Because this study works on typographic artifacts rather than human respondents, its "instruments" are the software apparatus that manufactures, measures, and adjudicates the data rather than a questionnaire or survey. Four instruments are used: a researcher-made synthetic-data generation pipeline that produces the training and validation corpus, a set of standardized evaluation metrics that score the model's output, a researcher-made human-proxy consensus rubric that fixes ground truth on real generative crops, and a web application that exercises the trained model in real time. The first three supply and assess data; the fourth is the interactive artifact through which the system is demonstrated and stress-tested.

### 4.2.1 Synthetic-Data Generation Pipeline (Researcher-Made)

The primary instrument is a researcher-built rendering-and-degradation pipeline. Its purpose is to generate a large, fully labeled corpus of text-crop images for which no in-the-wild equivalent exists at the scale metric learning requires, following the synthetic-corpus precedent of DeepFont (Wang et al., 2015) and the more recent DINOv2-based font pipeline of Chen et al. (2026). The instrument has two stages. The rendering stage draws pristine word crops from the selected Google Fonts palette; the degradation stage applies the operator $D$ defined in Section 3.2.3 (elastic warp, Gaussian blur and noise, and kerning jitter) to convert each pristine crop into a hallucinated variant.

The instrument's **construct validity** rests on the correspondence between its degradation parameters and the deformation mechanisms the generative literature documents: cross-character attention smearing (Gillani et al., 2025), multi-instance entanglement (Du et al., 2025), and continuous latent-manifold mutation (Kondo et al., 2024). In other words, the pipeline is valid to the extent that its manufactured deformations reproduce those observed in real text-to-image output. Its **reliability** is guaranteed procedurally: rendering and degradation are deterministic given a fixed random seed, so any crop in the dataset can be regenerated bit-for-bit, and the per-parameter degradation levels are logged with each image. This distinguishes the instrument from a standardized dataset, since it is researcher-made, but its determinism makes it reproducible by any third party who runs the same code with the same seed.

### 4.2.2 Standardized Evaluation Metrics

The model's output is scored with published, standardized metrics rather than researcher-defined ones, which imports their established validity. Recognition accuracy uses the **Top-1 and Top-3 (Top-K) protocol** that DeepFont established as the field standard (Wang et al., 2015). Typographic proximity between a predicted font and the input crop is measured with **structural-similarity metrics** in a re-render-and-compare loop: the classical SSIM of Wang et al. (2004) and its deformation-robust successors DeepSSIM (Zhang et al., 2024) and SAMScore (Li, Chen, et al., 2025), whose reliability under geometric misalignment is exactly the property this study needs. Prediction-set confidence uses **rank-calibrated conformal prediction** (Shi et al., 2024; Ding et al., 2025), which carries a formal coverage guarantee. Finally, error severity is weighted by the **embedding-centroid cosine severity index** of Chen et al. (2026), which quantifies whether misclassifications stay within a typographic family. Because these instruments are drawn from peer-reviewed sources, their validity and reliability coefficients are inherited from the literature rather than re-established here.

### 4.2.3 Human-Proxy Consensus Rubric (Researcher-Made)

Ground truth for real generative crops, which carry no font metadata, is fixed by a researcher-made rating instrument administered to a three-person expert panel. The instrument is delivered asynchronously as a web-based rating form rather than in a supervised session (the remote environment justified in Section 4.1): each crop is presented individually beside a reference sheet of the palette rendered as labeled specimens, so that raters match against a fixed, visible option set instead of recalling fonts from memory, and each rater works self-paced and blind both to the other raters and to the model's predictions, which guards against anchoring. The form's two screens, an instructions page and the per-crop rating item, are reproduced in the appendices; the layout is deliberately platform-neutral, so any standard web-form service can deliver it without altering the instrument. For each of the [N = 100] real generative-AI text boxes, each rater independently assigns a font label from the palette or an "unknown / hallucinated" verdict, and a label is fixed only when at least two of three raters agree. The instrument's **content validity** comes from panel expertise (raters screened for typographic training, criteria specified in Section 4.10); its **reliability** is quantified by inter-rater agreement, reported as a Fleiss' $\kappa$ of [κ = X.XX] once the rating exercise is run. The 2-of-3 consensus rule is the instrument's built-in reliability safeguard against any single rater's idiosyncrasy.

### 4.2.4 Web Application (Interactive Testing Instrument)

The fourth instrument is a web application built on top of the trained model so that the system can be exercised in real time rather than only in batch evaluation. A user uploads a generative-AI image through the browser, the backend runs the inference engine, and the interface returns the Top-K Google Fonts shortlist or an "unknown" verdict. The accepted input is a raster image in PNG (.png), JPEG (.jpg, .jpeg), or WebP (.webp) format up to 10 MB: these are the formats current text-to-image generators export and the forms in which their output circulates as screenshots and web re-shares. Vector formats such as SVG are refused because typographic hallucination is a raster artifact with no vector equivalent, and animated or photographic formats (GIF, HEIC) fall outside the study's scope of directly generated still images. This instrument serves two methodological purposes: it is the demonstration artifact required by the build-method evaluation of Section 4.10, and it is a qualitative probe that surfaces failure cases on ad hoc, in-the-wild inputs that the fixed test set may not contain. Its design, framework choices, and development process are detailed in Sections 4.5 through 4.8.
